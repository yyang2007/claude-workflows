# 轮子原则

每个模块必须满足三个条件，否则不算完成。

## 三条件

1. **圆的** — 同输入同输出。边界清晰，不泄漏职责。LLM 等不稳定组件必须包 timeout + retry + fallback 外层。
2. **转一圈** — 感知输入→决策→执行→验证完成。输出必须经过自校验（字段非空、类型正确），print/log 不算完成。
3. **持续转** — 状态持久化到 DB/文件/localStorage，不挂内存。崩溃能续。每圈独立，不累积误差。

## 模块大小：能画一条线吗

```
输入 → 处理 → 输出
```

能在纸上画出来就是轮子。画不出来（还需要人盯着补一步）就是太大，继续拆。

## LLM 外包模式

LLM 不"圆"，但必须包圆了再暴露：

```
输入 → [ timeout + retry + schema校验 ] → LLM（内层波动） → [ fallback ] → 同输入=同输出或同Error
```

没有 timeout、没有 fallback 的 LLM 调用不算轮子。

## 不共享变量

轮子之间只通过输入输出连接。改 A 炸 B = 没拆干净。

## 三步快速检查

1. 数据流画得出来吗？
2. 它自己知道做完了吗（自校验）？
3. 下次还能这样跑吗（持久化）？
